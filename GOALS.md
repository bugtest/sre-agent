# GOALS.md - 项目目标和 OKR

## 项目愿景
构建一个高质量、可维护的软件系统，通过多代理协作实现快速迭代和持续交付。

---

## OKR 2026-Q1

### O1: 建立高效的多代理协作流程
- **KR1**: 完成 Architect/Dev/Test 三个代理的配置和联调
- **KR2**: 实现从需求到上线的完整自动化流程
- **KR3**: 平均交付周期从 3 天缩短到 1 天

### O2: 保证代码质量
- **KR1**: 测试覆盖率 ≥ 80%
- **KR2**: 严重 Bug 数 = 0
- **KR3**: Code Review 覆盖率 100%

### O3: 提升开发效率
- **KR1**: 减少 50% 的重复性工作
- **KR2**: 需求理解偏差导致的返工 < 10%
- **KR3**: 设计文档覆盖率 100%

---

## 当前优先级

1. **P0**: 完成项目基础设置
2. **P1**: 跑通第一个完整工作流（需求→设计→开发→测试）
3. **P2**: 优化代理协作效率

---

## 技术原则

1. **设计先行**: 任何功能实现前必须有设计文档
2. **测试驱动**: 代码必须附带测试
3. **小步快跑**: 任务拆解到可独立验证的粒度
4. **透明协作**: 所有状态变更写入 STATE.yaml
5. **质量门禁**: 测试不通过不合并
